Overview

Vim is a highly configurable text editor for efficiently creating and changing any kind of text. Vim is a clone, with additions, of Bill Joy's vi text editor program for Unix. Vim's author, Bram Moolenaar, based it upon the source code for a port of the Stevie editor to the Amiga and released a version to the public in 1991.
 
PavedRoad has enhanced Vim to be a complete IDE adding word completion and development related shell commands that can be run inside of Vim. These commands are mapped to keystrokes for execution allowing developers to work without ever having to leave Vim. If the version of Vim on the target device is not 8.0 or later then the latest version of will be installed.

Plugins
PavedRoad has integrated 5 vim plugins together. These five plugins provide a developer with an enhanced VIM environment that requires minimal effort on the developers part to get setup and running. The plugins called vim-go and deoplete-go supply respectively the vim go commands and vim go completion.
 
  1. vim-go - provides Go language support for VIM.
  2. deoplete-go - asynchronous Go completion for Neovim. deoplete source for Go. deoplete.nvim is a required plugin for deoplete-go
  3. deoplete.nvim - Deoplete is the abbreviation of "dark powered neo-completion". Deoplete provides an extensible and asynchronous completion framework for neovim/Vim8. deoplete will display completions via complete() by default.
  4. nvim-yarp - is a remote plugin framework for Neovim without ':UpdateRemotePlugins' and is a required plugin for deoplete.nvim plugin.
  5. vim-hug-neovim-rpc - Used to build a compatibility layer for neovim rpc client working on vim8. - required plugin for deoplete.nvim plugin.
For now we rely on the default settings for the plugins. More information about the vim-go settings can be located in section 7 of its document page. All of the settings for deoplete-go are described in the Available Settings section of its main landing page.
