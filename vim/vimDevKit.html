DevKit

Overview

While Vim is technically not an IDE, it is the most popular text editor used by developers today, which is why we at PavedRoad selected it. However, it lacks some of the capabilities found in other IDEs such as Microsoft Visual Studio, therefore, we have integrated plugins to add the missing capabilities such as syntax completion, language documentation, compiling and debugging in the editor directly. To add these capabilities in Vim provides a plugin system. From that system, we have selected and integrated 5 plugins.

PavedRoad enhancements

Plugins
PavedRoad has incorporated 5 plugins in Vim. These plugins provide a developer with an enhanced Vim environment by providing the convenience of the most typical Go commands. This environment requires minimal effort on the developers part to get set up and running. The plugins called vim-go and deoplete-go supply respectively the vim go commands and vim go completion.
 
Plugin Description
vim-go
This plugin adds Go language support for Vim with the following features: code completion, snippet support, improved syntax highlighting, go toolchain commands. It also provides the ability to compile, and test as well as execute and debug files plug much much more. Individual features can be toggled easily. vim-go leverages a number of tools developed by the Go community to provide a seamless Vim experience.
deoplete-go deoplete.nvim
This plugin will display completions via complete() by default for Go. The plugins nvim-yarp and vim-hug-neovim-rpc for Vim8 are required for deoplete-go PavedRoad has also installed those by default. All of the settings for deoplete-go are described in the available settings section of its main landing page.
nvim-yarp
Provides a remote plugin framework for Neovim without ':UpdateRemotePlugins' and is a required plugin for deoplete.nvim plugin.
vim-hug-neovim-rpc
Used to build a compatibility layer for neovim rpc client working on vim8. - required plugin for deoplete.nvim plugin
For now we rely on the default settings for the plugins. More information about the vim-go settings can be located in section 7 of its document page.
